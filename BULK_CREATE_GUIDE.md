## 💡 Pro Tips

### Preparation
1. **Use a text editor:** Write your task list in VS Code or Sublime first
2. **Keep it simple:** Don't include type/priority prefixes - just the task name
3. **Format nicely:** One task per line, remove extra spaces
4. **Review before bulk creating:** Make sure you have the right config

### Best Practices
1. **Start empty:** Create with "Unassigned" and assign in bulk after
2. **Use default priority:** Use P2 or P3 as default, adjust outliers later
3. **Batch by type:** All features together, all bugs together
4. **Name consistently:** Use similar naming patterns for easier bulk operations

### After Creation
1. **Bulk assign:** Filter by status BACKLOG, then bulk select and assign
2. **Set deadlines:** Use the full task editor to set deadlines later
3. **Add descriptions:** Add details to complex tasks individually
4. **Link tasks:** Connect related tasks as parent/child relationships

## ❓ FAQ

**Q: Can I create tasks with descriptions using Bulk Create?**
A: No, bulk create is for names only. Use Full Create for tasks needing descriptions.

**Q: What if some tasks fail to create?**
A: You'll see which ones succeeded and which failed. No partial failures - you get feedback for each one.

**Q: Can I set different priorities for each task?**
A: Not in bulk create. Create them all with one priority, then bulk edit to adjust.

**Q: How many tasks can I create at once?**
A: There's no hard limit! We've tested up to 100 tasks. Go wild! 🚀

**Q: Are the Type/Priority/Assignee fields remembered?**
A: Yes! Your last bulk create settings are saved and auto-fill next time.

**Q: Can I undo a bulk creation?**
A: Yes! Delete the created tasks individually or use bulk delete (if available).
